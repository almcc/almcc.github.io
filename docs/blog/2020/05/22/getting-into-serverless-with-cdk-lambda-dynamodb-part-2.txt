e:T3644,<p>In this second post, we are going to add a DynamoDB Table to our stack for our lambda function to pull data from. See <a href="Getting%20into%20Serverless%20with%20CDK,%20Lambda%20&#x26;%20DynamoD%20dcfbc72d480e4c70b0ae73884c901386.md">Getting into Serverless with CDK, Lambda &#x26; DynamoDB - Part 1</a>  on getting started to get caught up.</p>
<p>Before we go too far, you are going to need:</p>
<ul>
<li>Python 3.8 with Pip installed</li>
<li><a href="https://docs.aws.amazon.com/serverless-application-model/latest/developerguide/serverless-sam-cli-install.html">AWS SAM CLI</a></li>

<h2>Step 1 - Defining the table.</h2>
<p>Install the DynamoDB CDK module, being careful to use the same release you used before.</p>
<pre><code class="hljs language-css">npm install <span class="hljs-keyword">@aws-cdk</span>/aws-dynamodb<span class="hljs-keyword">@1</span>.33.1
</code></pre>
<p>And add it to our imports in <code>lib/myapp-stack.ts</code>.</p>
<pre><code class="hljs language-tsx"><span class="hljs-keyword">import</span> * <span class="hljs-keyword">as</span> dynamodb <span class="hljs-keyword">from</span> <span class="hljs-string">"@aws-cdk/aws-dynamodb"</span>;

</code></pre>
<p>Next, we can add a DynamoDB <a href="https://docs.aws.amazon.com/cdk/api/latest/docs/@aws-cdk_aws-dynamodb.Table.html">Table</a> construct to our stack. DynamoDB is a no-sql database and requires you to think a little bit about how your data will be accessed. Read about the key concepts over <a href="https://www.dynamodbguide.com/key-concepts/">here</a> on <a href="https://www.dynamodbguide.com/">dynamodbguide.com</a></p>
<pre><code class="hljs language-tsx"><span class="hljs-keyword">const</span> tourfinishersTable = <span class="hljs-keyword">new</span> dynamodb.<span class="hljs-title class_">Table</span>(<span class="hljs-variable language_">this</span>, <span class="hljs-string">"TourFinishersTable"</span>, {
  <span class="hljs-attr">partitionKey</span>: { <span class="hljs-attr">name</span>: <span class="hljs-string">"Year"</span>, <span class="hljs-attr">type</span>: dynamodb.<span class="hljs-property">AttributeType</span>.<span class="hljs-property">NUMBER</span> },
  <span class="hljs-attr">sortKey</span>: { <span class="hljs-attr">name</span>: <span class="hljs-string">"Position"</span>, <span class="hljs-attr">type</span>: dynamodb.<span class="hljs-property">AttributeType</span>.<span class="hljs-property">NUMBER</span> },
  <span class="hljs-attr">tableName</span>: <span class="hljs-string">"TourFinishers"</span>,
  <span class="hljs-attr">billingMode</span>: dynamodb.<span class="hljs-property">BillingMode</span>.<span class="hljs-property">PAY_PER_REQUEST</span>,
  <span class="hljs-attr">removalPolicy</span>: cdk.<span class="hljs-property">RemovalPolicy</span>.<span class="hljs-property">DESTROY</span>,
});
</code></pre>
<p>We have chosen in this instance to make the <code>Year</code> the primary key and <code>Position</code> the sort key, together this will be a unique value within this simple context a high cardinality. It will support our access pattern later as will.</p>
<p>Next step is to grant our function read access to the table.</p>
<pre><code class="hljs language-tsx">tourfinishersTable.<span class="hljs-title function_">grantReadData</span>(myFunction);
</code></pre>
<p>I think this is one of the most powerful features of CDK, it makes dealing with permissions so easy that you don't feel the need to grant overly permissive policies.</p>
<h2>Step 2 - Updating out function</h2>
<p>Our python function is going to be a little more complex than before and will require some dependencies. Therefore we first need to move it into its own <code>src</code> directory.</p>
<pre><code class="hljs language-bash"><span class="hljs-built_in">mkdir</span> <span class="hljs-keyword">function</span>/src
<span class="hljs-built_in">mv</span> <span class="hljs-keyword">function</span>/index.py <span class="hljs-keyword">function</span>/src/
</code></pre>
<p>Next, we will want to add a requreiments.txt file at <code>function/src/requirements.txt</code> and add the following dependencies.</p>
<pre><code class="hljs language-shell"><span class="hljs-meta prompt_">boto3></span><span class="bash">=1.12.43</span>
</code></pre>
<p>Now we will write a <code>build.sh</code> script at the base of our project to build are code bundle. AWS Lambda's require a code bundle to come pre-packaged with all its dependencies.</p>
<pre><code class="hljs language-bash"><span class="hljs-meta">#!/bin/bash</span>

<span class="hljs-built_in">mkdir</span> -p <span class="hljs-keyword">function</span>/_bundle
pip3.8 install -r <span class="hljs-keyword">function</span>/src/requirements.txt --target <span class="hljs-keyword">function</span>/_bundle
<span class="hljs-built_in">cp</span> -r <span class="hljs-keyword">function</span>/src/* <span class="hljs-keyword">function</span>/_bundle
<span class="hljs-built_in">pushd</span> <span class="hljs-keyword">function</span>/_bundle
zip -r --quiet ../bundle.zip .
<span class="hljs-built_in">popd</span>
<span class="hljs-built_in">rm</span> -rf <span class="hljs-keyword">function</span>/_bundle
</code></pre>
<p>And now we can update our code <code>function/src/index.py</code> to pull the data from dynamoDB.</p>
<pre><code class="hljs language-python"><span class="hljs-keyword">import</span> boto3

<h2>Bonus Step - Running the function locally</h2>
<p>You can also run the function locally with the AWS sam CLI which can be handy during development.</p>
<p>The first thing to do is to get hold of the CloudFormation template that CDK is creating behind the scenes for us.</p>
<pre><code class="hljs language-arduino">cdk synth --no-staging > <span class="hljs-keyword">template</span>.yaml
</code></pre>
<p>Now you can use the <code>sam</code> CLI to invoke your function. (Check the contents of <code>template.yaml</code> for the functions name)</p>
<pre><code class="hljs language-lua">sam <span class="hljs-keyword">local</span> invoke MyFunction3BAA72D1 <span class="hljs-comment">--no-event | jq</span>
</code></pre>
<p>This will run your code locally, using your local credentials to query the live table in AWS.</p>
<p>Then if you make any changes to you code you will need to run <code>./build.sh</code> again, if you make a change to your template, you will need to do the following.</p>
<pre><code class="hljs language-arduino">npm run build
cdk synth --no-staging > <span class="hljs-keyword">template</span>.yaml
</code></pre>
