2:Te26,<p>For any software project it is likely that you're will need to define the version in sperate place and keeping track of that can be a bit of pain. <a href="https://github.com/peritus/bumpversion">bumpversion</a> is a very handy tool to help you keep track. You can install it like so:</p>
<pre><code class="hljs language-css">pip install <span class="hljs-attr">--upgrade</span> bumpversion
</code></pre>
<p>Taking a python project for example, for me the version will be defined in <code>setup.py</code>:</p>
<pre><code class="hljs language-python"><span class="hljs-keyword">from</span> setuptools <span class="hljs-keyword">import</span> setup

setup(
    name=<span class="hljs-string">'example'</span>,
    version=<span class="hljs-string">'0.0.1'</span>,
    packages=[<span class="hljs-string">'example'</span>],
)
</code></pre>
<p>and <code>example/__init__.py</code> (Package called <code>example</code> here.):</p>
<pre><code class="hljs language-python">VERSION = <span class="hljs-string">'0.0.1'</span>
</code></pre>
<p>To setup bumpversion simple create <code>.bumpversion.cfg</code>:</p>
<pre><code class="hljs language-ini"><span class="hljs-section">[bumpversion]</span>
<span class="hljs-attr">current_version</span> = <span class="hljs-number">0.0</span>.<span class="hljs-number">1</span>
<span class="hljs-attr">commit</span> = <span class="hljs-literal">True</span>
<span class="hljs-attr">tag</span> = <span class="hljs-literal">True</span>

<span class="hljs-section">[bumpversion:file:setup.py]</span>

<span class="hljs-section">[bumpversion:file:example/__init__.py]</span>

</code></pre>
<p>Here we are recording the <code>current_version</code> which is used to do the find and replace in our files, <code>commit = True</code> and <code>tag = True</code> is saying if we are in a git repo, commit and tag the repo. If you are in a git repo the action will fail if the repo is not clean.</p>
<p>With bumpversion you can specify the new version, but I prefer to just specify the part to bump, aka <code>major|minor|patch</code>. So actually bumping the version is done like so:</p>
<pre><code class="hljs">bumpversion patch
</code></pre>
<p>for a patch bump. If you just want to see what would happen you can do the following:</p>
<pre><code class="hljs language-css">bumpversion <span class="hljs-attr">--dry-run</span> <span class="hljs-attr">--allow-dirty</span> <span class="hljs-attr">--verbose</span> patch
</code></pre>
<p>As I'm usually using <a href="http://www.pyinvoke.org/">Python Invoke</a>, I create the following task:</p>
<pre><code class="hljs language-python"><span class="hljs-keyword">from</span> invoke <span class="hljs-keyword">import</span> task

<span class="hljs-meta">@taskdef bump_version(<span class="hljs-params">ctx, part, confirm=<span class="hljs-literal">False</span></span>):</span>
    <span class="hljs-keyword">if</span> confirm:
        ctx.run(<span class="hljs-string">'bumpversion {part}'</span>.<span class="hljs-built_in">format</span>(part=part))
    <span class="hljs-keyword">else</span>:
        ctx.run(<span class="hljs-string">'bumpversion --dry-run --allow-dirty --verbose {part}'</span>.<span class="hljs-built_in">format</span>(part=part))
        <span class="hljs-built_in">print</span>(<span class="hljs-string">'Add "--confirm" to actually perform the bump version.'</span>)
</code></pre>
<p>Therefore to do a quick comfort check:</p>
<pre><code class="hljs">invoke bump-version patch
</code></pre>
