e:Tfa9,<p>I have struggled for a while on how to handle my python requirements, fighting on one side to pin my requirements to exact versions to have repeatable builds, and on the other side to keep running the latest packages. It generally ended up as a mess.</p>
<p>However I have just stumbled across <a href="https://github.com/jazzband/pip-tools">pip-tools</a>, it allows me to keep packages pinned to exact versions so that a single git commit builds the exact same image every time, yet at the same time offering the tools to easily update those packages to the latest ones available.</p>
<p>To get started just create a virtual environment:</p>
<pre><code class="hljs language-bash">python3.6 -m venv venv
. ./venv/bin/activate
</code></pre>
<p>and install pip-tools with:</p>
<pre><code class="hljs language-bash">pip install pip-tools
</code></pre>
<p>I like to create two requirements files, the first requirements.txt for the bare minimum to run the application, next dev-requirements.txt which includes everything else to be able to develop on the application. To use pip-tools, rather than creating <code>.txt</code> files, we create <code>.in</code> files where you list you package names, you can pin them here if you app requires a particular version, but if it doesn't just omit the version number.</p>
<p>Create your <code>requirements.in</code> and <code>dev-requirements.in</code> files:</p>
<pre><code class="hljs language-bash"><span class="hljs-built_in">echo</span> Django > requirements.in
<span class="hljs-built_in">echo</span> invoke > dev-requirements.in
</code></pre>
<p>Now you can use pip-compile to "compile" your <code>.in</code> files into the typical python
requirements <code>.txt</code> files:</p>
<pre><code class="hljs language-bash">pip-compile requirements.in
pip-compile dev-requirements.in
</code></pre>
<p>And now you have fully pinned requirements files that you should commit to source control along with your <code>.in</code> files.</p>
<p>You can of course now run <code>pip install -r requirements.txt -r dev-requirements.txt</code> however I would suggest you use <code>pip sync</code> which is part of pip-tools. It will not only install the required packages, but also remove packages not defined in your requirements files:</p>
<pre><code class="hljs language-bash">pip-sync dev-requirements.txt requirements.txt
</code></pre>
<p>When you want to update your pinned packages all you need to do is run:</p>
<pre><code class="hljs language-bash">pip-compile --upgrade requirements.in
pip-compile --upgrade dev-requirements.in
</code></pre>
<p>and your <code>.txt</code> files will be update to reflect the latest packages available on <a href="https://pypi.org/">pypi</a>.</p>
<p>I usually also have a <a href="http://www.pyinvoke.org/">Python Invoke</a> <code>tasks.py</code> file for my projects, I have added the follow two commands to help me with syncing my virtual environment and also updating my requirements.</p>
<pre><code class="hljs language-python"><span class="hljs-keyword">from</span> invoke <span class="hljs-keyword">import</span> task

<span class="hljs-meta">@task</span>
<span class="hljs-keyword">def</span> <span class="hljs-title function_">sync_venv</span>(<span class="hljs-params">ctx</span>):
    <span class="hljs-string">"""Sync the local venv with requirements."""</span>
    ctx.run(<span class="hljs-string">'pip-sync dev-requirements.txt requirements.txt'</span>)

<span class="hljs-meta">@task</span>
<span class="hljs-keyword">def</span> <span class="hljs-title function_">update_requirements</span>(<span class="hljs-params">ctx</span>):
    <span class="hljs-string">"""Update the requirements.txt files to the latest packages."""</span>
    ctx.run(<span class="hljs-string">'pip-compile --upgrade requirements.in'</span>)
    ctx.run(<span class="hljs-string">'pip-compile --upgrade dev-requirements.in'</span>)
</code></pre>
